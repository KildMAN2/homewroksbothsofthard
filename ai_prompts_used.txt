Prompt 1: "Create a C++ DNA motif evolution simulation — naive version with branch-heavy scoring and one optimized using a precomputed LUT and rolling window. Same I/O and checksum for both."

Prompt 2: "Write a shell script to compile, correctness-check, and perf-profile both versions with hardware/software counter fallback."

Prompt 3: "Help write a 3-page HW1 report covering naive design, LUT+rolling-window optimization, perf comparison, and the hardware reasoning behind the 15.7x speedup."

Optimizations applied and understood:
- LUT: precomputes all 1024 possible 5-mer scores, eliminating 7 data-dependent branches per element.
- Rolling window: maintains the 10-bit neighborhood incrementally (shift+mask) instead of 5 separate indexed loads per step.
- Hardware insight: branch mispredictions on random-like data cost ~15-20 pipeline cycles each; removing branches entirely via LUT recovered that lost IPC.
